In this article, we'll look at the @DataJpaTest for JPA testing among the data layer slice test annotations, and we'll also discuss the related @AutoConfigureTestDatabase.

@DataJpaTest

This annotation is used to focus the test on the JPA component, disabling the entire autoconfiguration and instead applying only the configuration relevant to the JPA test.

It is important to understand that because @Transactional exists inside @DataJpaTest, the data will be rolled back even if the test runs successfully. Also, because @AutoConfigureTestDatabase exists inside @DataJpaTest, the datasource related settings will be ignored and the embedded database will be used.

showSql

This attribute acts like spring.jpa.show-sql and specifies whether to print SQL logs. The default value is true.

bootstrapMode

Specifies the bootstrap mode, meaning when the repository beans are initialized. In the example below, we'll replace bootstrapMode and see how the logs change.

@DataJpaTest(bootstrapMode = BootstrapMode.DEFAULT)
class TestUsingDataJpaTest {
    @Autowired
    private MemberRepository memberRepository;
    @Test
    void createMemberTest() {
        Member member = new Member();
        member.setAge(10);
        member.setUsername("foo");
        memberRepository.save(member);
    }
}

properties

It is a key=value attribute that needs to be added to the Spring environment before running the test.

useDefaultFilters

This attribute specifies whether the filters used in the @SpringBootApplication should also be applied to tests. Defaults to true and does not include any beans.

includeFilters

When used, only component classes that meet certain conditions can be registered as beans. This means that you can filter the component classes to be registered as beans based on certain types, annotations, etc.

@DataJpaTest(
  includeFilters = @ComponentScan.Filter(
    type = FilterType.ANNOTATION,
    classes = {Controller.class, Service.class}
  )
)
class DataLayerTest { ... }

excludeFilters

As opposed to the includeFilters attribute, this attribute is for filtering out component classes that are not to be registered as beans.

@DataJpaTest(
  excludeFilters = @ComponentScan.Filter(
    type = FilterType.ASSIGNABLE_TYPE,
    classes = {GenericRepository.class}
  )
)
class DataLayerTest { ... }

excludeAutoConfiguration

This attribute is used when you want to exclude certain autoconfiguration classes. In this example, we're excluding Flyway from the autoconfiguration.

@DataJpaTest(excludeAutoConfiguration = FlywayAutoConfiguration.class)
class DataLayerTest { ... }

@AutoConfigureTestDatabase

Use to configure a test database to be used in place of an application-defined or automatically configured datasource. Applies only to the @Primary datasource if there are multiple datasource beans. It can also be combined with @SpringBootTest.

replace

Specifies the conditions to replace with the embedded database for testing.

  • ANY

This is the default value and replaces the datasource bean regardless of whether the datasource bean is automatically configured or manually defined.

  • AUTO_CONFIGURED

Replace only if the datasource is automatically configured because you did not make any specific datasource settings.

  • NONE

In no case use an embedded database.

connection

This attribute specifies which embedded database should be used as the database for testing.

  • NONE

This is the default value and attempt detection by accessing the classpath.

  • H2

Specify the H2 database.

  • DERBY

Specifies the DERBY database.

  • HSQLDB

Specify the HSQLDB database.
