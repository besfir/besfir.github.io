Database design is so difficult that when I tried to organize it, I realized that I don't know anything about database basics. So, before I start writing the Database Design series, I'm going to summarize the Getting Started with Databases series. In this article, I'll summarize the information you need to get started with databases.

Database

In general, it is an organized set of related data that is stored and used electronically.

Related data

Consider a social networking service as an example. Users post photos, write posts, comment on other users' content, and more. As such, data that has the same origin and purpose within the same service is called related data.

Organized set

Imagine that your books are strewn across the floor. It's hard to find them quickly, and it's hard to tell if you have duplicates of the same book. But if you organize them and put them on a bookshelf, you can quickly find the book you want and easily see if you have unnecessary duplicates. A set that organizes data in this way to increase efficiency and data accessibility is called an organized set.

Stored and used electronically

It means storing data efficiently and keeping it on a computer where it can be quickly retrieved. This allows for fast computation and retrieval while taking up little space.

DBMS

Software that provides users with the ability to define, create, and manage databases, most commonly PostgreSQL, MySQL, Oracle, and MsSQL. In the process of defining a database, the structure of the data is determined, and this process generates additional data, which is called metadata.

Metadata

The term is widely used in a variety of fields, and its original meaning is "data about data," meaning that it refers to data that describes or characterizes data. For example, when you take a photo, you may see additional information that describes the photo, such as the date it was taken, the file format, the resolution, etc.

In the database world, it refers to the data that defines or describes a database, which is sometimes referred to as a catalog. The catalog can refer to the metadata itself, but it can also refer to the place where the metadata is stored and managed by the DBMS.

Examples of metadata include data types, structures, constraints, security, storage, indexes, and user groups.

Database system

it refers to the Database + DBMS + associated applications. However, if you read several documents, you may shorten the database system and call it just database. Therefore, it is necessary to understand the meaning by carefully grasping the context of whether the database of the document means pure data or database system.

Workflow

  • Users / Programmers: The application is used by users or programmers.
  • Application Programs / Queries: The application sends queries to the DBMS to perform some function requested by the user. In this context, a query is a request to get the desired data from the database, or to modify or delete it in a desired form.
  • Software to Process Queries / Programs: Analyze what the query means.
  • Software to Access Stored Data: Once it understands what is being requested, it processes the request.
    • Stored Database Definition: DBMS needs to know the metadata before we can process the data, so we find and verify the metadata first.
    • Stored Database: Process the desired data based on the metadata and return the results to the application.

Data Models

A set of concepts that can be used to describe the structure of a database. Based on those concepts, it provides a way to abstract the structure of a database and represent it in a desired form. The data model also includes the basic operations for reading and writing to the database. The database structure mentioned refers to concepts that include data types, relationships between data, constraints, etc.

There are several types of data models, each with a slightly different level of abstraction and a different way of structuring the database.

Conceptual (or high-level) data models

It is a model consisting of concepts that can be easily understood by not only developers but also ordinary users, and has the highest level of abstraction. This data model is used to describe business requirements at an abstract level. A representative data model is the Entity-Relationship Model, and its visual representation is the Entity-Relationship Diagram, or ER Diagram for short.

logical (or representational) data models

It provides concepts that allow you to structure your database in a way that is not too difficult to understand. Structuring the database in detail means structuring the logical data model so that it is not significantly different from the structure of the data model stored on the computer. However, this model does not even represent how data is stored on a physical storage device, but rather is a model that can structure the database at a level that is not dependent on a specific DBMS or storage.

The main data model is the relational data model, and other data models include the object data model and the object-relational data model. Among the major DBMSs mentioned above, Oracle, MySQL, and MsSQL are based on the relational data model. And PostgreSQL is based on an object-relational data model but includes a relational data model, so it can be said that the major DBMSs are based on a relational data model.

In the Getting Started with Databases series, we'll focus on this relational data model.

As can be seen in the figure above, relationship simply means a table. In a table, row means one data and is called a tuple or record. In addition, a column is called an attribute and refers to the characteristics of a tuple or record.

physical (or low-level) data models

It provides a way to describe how data is stored in files on a computer. In other words, it is a data model that expresses elements such as data format, data orderings, and access paths to represent the database structure as closely as possible to the way data is actually stored on the storage device. Among them, access paths are structures to speed up data retrieval, such as indexes.
