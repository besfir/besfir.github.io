Using and customizing the Tailwind CSS Typography Plugin

The @tailwindcss/typhography plugin is used to add typography defaults to HTML elements that do not allow you to specify a Tailwind CSS class directly, such as HTML rendered from markdown or imported from a CMS. You can override them directly in the CSS file without using this plugin, but if you do, the CSS classes you set may not be overridden based on class priority. However, the advantage of using this plugin is that it prioritizes the styles specified by the developer.

Install

pnpm i -D @tailwindcss/typography
tailwind.config.js
/** @type {import('tailwindcss').Config} */
module.exports = {
  theme: {
    // ...
  },
  plugins: [
    require('@tailwindcss/typography'),
    // ...
  ],
}

Basic Usage

Adding the plugin does not immediately apply globally, but only to the element that added the prose class. To help you understand, here's a snippet from the demo provided by Tailwind CSS.

<article class="prose">
  <h1>Garlic bread with cheese: What the science tells us</h1>
  <p>
    For years parents have espoused the health benefits of eating garlic bread with cheese to their
    children, with the food earning such an iconic status in our culture that kids will often dress
    up as warm, cheesy loaf for Halloween.
  </p>
  <p>
    But a recent study shows that the celebrated appetizer may be linked to a series of rabies cases
    springing up around the country.
  </p>
  <!-- ... -->
</article>
  • Because we added the prose class in line 1, the default typography style is applied to that element.

Choosing Themes

The @tailwindcss/typography plugin comes with five gray scale colors suitable for body text as modifier classes. This preset changes the color of all element modifiers, including body, headings, and links, so it deserves to be called a theme. Henceforth, we'll refer to it as a theme.

ClassGray scale
prose-gray (default)Gray
prose-slateSlate
prose-zincZinc
prose-neutralNeutral
prose-stoneStone

To use it, simply add the class to the desired element. As a word of caution, modifier classes are designed to be used in conjunction with prose classes and should always be used in conjunction with prose classes.

<article class="prose prose-stone">
  {{ markdown }}
</article>

If you're curious about the colors applied to elements in each theme, you can check out the code on GitHub

Choosing Size Modifier

The size modifier allows you to adjust the overall font size of your typography.

ClassBody font size
prose-sm0.875rem(14px)
prose-base1rem(16px)
prose-lg1.125rem(18px)
prose-xl1.25rem(20px)
prose-2xl1.5rem(24px)

For example, if you want to use a 16px font for a viewport size of 1024px or less, and an 18px font for larger viewport sizes, you can do something like this

<article class="prose lg:prose-lg">
  {{ markdown }}
</article>
  • The 16px font is omitted because it is the default.
  • We used the responsive modifier lg to ensure that prose-lg is applied on devices with a viewport size of 1024px or larger.
    • prose-lg has a body font size of 18px.

Apply dark mode

Each default color theme also includes a dark mode. So you don't need to configure anything, just add the class and use it.

<article class="prose dark:prose-invert">
  {{ markdown }}
</article>
  • Using the dark variant, we added a prose-invert class to use in dark mode.

Element Modifier

You can use element modifiers to customize styles for individual HTML elements. This makes it easy to change link colors, image borders, heading tag appearance, and more to match your brand style.

<article class="prose prose-img:rounded-xl prose-headings:underline prose-a:text-blue-600">
  {{ markdown }}
</article>
  • prose-img:rounded-xl: applyborder-radius` of 0.75rem to the image tag.
  • prose-headings:underline: Apply underline to heading tags such ash1, h2`...
  • prose-a:text-blue-600: Applies the color#2563ebto thea` tag.

Using Element Full Width

The prose class is limited to a maximum width of 65ch each for readability. If you want to use the full width, you can add a max-w-none class.

<div class="grid grid-cols-4">
  <div class="col-span-1">
    <!-- ... -->
  </div>
  <div class="col-span-3">
    <article class="prose max-w-none">
      {{ markdown }}
    </article>
  </div>
</div>

Advance Usage

Let's take a look at what you can do if you don't want to be affected by the typographic style within an element with the prose class applied, if you want to make some styling customization, and if you want to apply additional themes.

Excluding typography styles

The not-prose class prevents the typographic style from being affected within elements with the prose class added.

<article class="prose">
  <h1>Some Heading</h1>
  <p>...</p>

  <div class="not-prose">
    <!-- Elements that you don't want to apply a typographic style to. -->
  </div>

  <p>...</p>
  <!-- ... -->
</article>

Keep in mind that even if you write prose inside not-prose, you can't use the typography style.

Customize Styling

This is the purpose of this article. In order to customize the typography style, you need to enter values into the typography coloration according to predefined keys.

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [],
  theme: {
    extend: {
      typography: theme => ({
        invert: {
          css: {},
        },
        DEFAULT: {
          css: {},
        },
        sm: {
          css: {},
        },
        // ...
      }),
    },
  },
  plugins: [require("@tailwindcss/typography")],
}

The predefined keys are :

  • Size : sm, base, lg, xl, 2xl
  • Theme : slate, gray, zinc, neutral, stone, red, orange, amber, yellow, lime, green, emerald, teal, cyan, sky, blue, indigo, violet, purple, fuchsia, pink, rose,
  • Dark mode : invert

To use the above key values, type prose-[key]. You can check the GitHub code to see if each key has been styled.

Adding a new style

In addition to the defined key values, you can define your own new key values. For example, suppose you want to add the key value md.

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [],
  theme: {
    extend: {
      typography: theme => ({
        md: {
          css: {
            a: {
              color: theme('colors.red.500'),
            },
          },
        },
      }),
    },
  },
  plugins: [require("@tailwindcss/typography")],
}

<article class="prose lg:prose-md">
  {{ markdown }}
</article>
  • If the viewport size is larger than 1024px, prose-md is applied and the <a> tag is colored red.

I wrote a demo for this in Tailwind playground.
