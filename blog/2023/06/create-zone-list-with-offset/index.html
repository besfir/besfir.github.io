Extracting a list of time zones with Java +Kotlin

Besfir · June 14, 2023
Table of Contents
  • Code
  • Explanation
  • Kotlin code

I searched online because the service I was developing needed timezone and offset information, but I couldn't find the format I wanted. Therefore, processing was necessary, but I wanted to process it into a code so that it could be easily managed in the future. I remember seeing the API that brings up the list of zones on the time package side, so when I looked it up, I found getAvailableZoneIds(), and I succeeded in extracting the list using it, so I summarized it.

Code

Tested with Java 19

import java.time.LocalDateTime;
import java.time.ZoneId;
import java.time.ZoneOffset;
import java.util.Comparator;
class Scratch {
  public static void main(String[] args) {
    LocalDateTime now = LocalDateTime.now();
    ZoneId.getAvailableZoneIds().stream()
            .map(zoneId -> {
              ZoneId zone = ZoneId.of(zoneId);
              return new ZoneIdWithOffset(zoneId, zone.getRules().getOffset(now));
            })
            .sorted(Comparator.comparing(o -> o.offset().getTotalSeconds()))
            .forEach(System.out::println);
  }
}
record ZoneIdWithOffset(String zoneId, ZoneOffset offset) {
  @Override
  public String toString() {
    return String.format("%s (UTC%s)", zoneId, offset.toString().replace("Z", "+00:00"));
  }
}
Output
Etc/GMT+12 (UTC-12:00)
Pacific/Pago_Pago (UTC-11:00)
Pacific/Samoa (UTC-11:00)
Pacific/Niue (UTC-11:00)
US/Samoa (UTC-11:00)
...

Explanation

record ZoneIdWithOffset(String zoneId, ZoneOffset offset) {
  @Override
  public String toString() {
    return String.format("%s (UTC%s)", zoneId, offset.toString().replace("Z", "+00:00"));
  }
}

The reason for using replace for offset is that if it is +-0, it is written as Z instead of +-xx:xx like other offsets. Therefore, it was used to unify the form.

public static void main(String[] args) {
  LocalDateTime now = LocalDateTime.now();
  ZoneId.getAvailableZoneIds().stream()
          .map(zoneId -> {
            ZoneId zone = ZoneId.of(zoneId);
            return new ZoneIdWithOffset(zoneId, zone.getRules().getOffset(now));
          })
          .sorted(Comparator.comparing(o -> o.offset().getTotalSeconds()))
          .forEach(System.out::println);
}

Without sorting, it is randomly mixed, not in name order or offset order. Therefore, I ordered it in ascending order based on offset.

Kotlin code

It's not that different, but I've also written Kotlin code.

import java.time.LocalDateTime
import java.time.ZoneId
import java.time.ZoneOffset
data class ZoneIdWithOffset(val zoneId: String, val offset: ZoneOffset) {
    override fun toString(): String {
        return String.format("%s (UTC%s)", zoneId, offset.toString().replace("Z", "+00:00"))
    }
}
val now = LocalDateTime.now()
ZoneId.getAvailableZoneIds()
    .map { zoneId ->
        val zone = ZoneId.of(zoneId)
        ZoneIdWithOffset(zoneId, zone.rules.getOffset(now))
    }
    .sortedBy { (_, offset) -> offset.totalSeconds }
    .forEach { v: ZoneIdWithOffset -> println(v) }
