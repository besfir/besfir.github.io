이번 글에서는 모르더라도 테스트하는 데 지장은 없지만, 알면 테스트를 쉽게 하는 데 도움을 주는 정보들을 정리할 예정이다.

@ParameterizedTest

서로 다른 인수를 이용해서 테스트를 여러번 반복하기 위해 사용한다. 반복될 인수 값을 지정하는 소스 어노테이션과 함께 사용한다. 여러 소스 어노테이션이 있지만 @ValueSource, @CsvSource만 알아도 충분히 큰 도움이 된다.

@ValueSource

리터럴 값의 단일 배열을 인수로 지정한다. 지원하는 타입은 short, byte, char, int, long, float, double, boolean, String, Class이다.

@ParameterizedTest
@ValueSource(ints = {1, 2, 3, 4, 5})
void parameterizedTestExample(int candidate) {
    assertTrue(candidate < 10);
}

@CsvSource

쉼표로 구분된 값(Comma Separated Value)을 인수로 지정한다. 각 인수는 CSV 레코드를 나타낸다.

@ParameterizedTest
@CsvSource({"ninja,700", "assassin,850", "'happy time',1_100", "@user-01x0w,0xE3F" })
void csvSourceExample(String name, int score) { }
Output
[1] ninja, 700
[2] assassin, 850
[3] happy time, 1_100
[4] @user-01x0w, 0xE3F

💡 여기서부터 @CsvSource의 바뀐 속성값에 집중할 수 있도록 필요없는 부분은 생략하겠다.

useHeadersInDisplayName 속성을 이용해서 첫번째 레코드를 선택적으로 CSV 헤더로 사용할 수 있다. true로 설정하면 첫번째 레코드를 헤더로 인식해서 스킵하고 Display name으로 표시한다.

useHeadersInDisplayName
@CsvSource(
    value = {"NAME,SCORE", "'happy time',1_100", "@user-01x0w,0xE3F"},
    useHeadersInDisplayName = true
)
Output
[1] NAME = happy time, SCORE = 1_100
[2] NAME = @user-01x0w, SCORE = 0xE3F

기본 구분자는 콤마(,)이다. delimiter 속성을 통해서 Char 타입 구분자를 지정할 수 있으며, delimiterString 속성을 통해서 String 타입 구분자를 지정할 수 있다. 두 속성을 함께 사용하면 PreconditionViolationException이 발생한다.

delimiter
@CsvSource(
    value = { "'happy time':1_100", "@user-01x0w:0xE3F" }, 
    delimiter = ':'
)
Output
[1] happy time, 1_100
[2] @user-01x0w, 0xE3F

따옴표 문자로 작은 따옴표(')를 기본으로 사용한다. 따옴표 문자로 묶인 빈문자열은 emptyValue 속성에 별도 값을 설정하지 않는다면 빈문자열을 반환한다.

quoteCharacter
@CsvSource(
    value = { "\"happy time\",1_100", "@user-01x0w,0xE3F" }, 
    quoteCharacter = '\"'
)
Output
[1] happy time, 1_100
[2] @user-01x0w, 0xE3F
empty string
@CsvSource({"'',1_100", "@user-01x0w,0xE3F" })
Output
[1] , 1_100
[2] @user-01x0w, 0xE3F
emptyValue
@CsvSource(
    value = { "'',1_100", "@user-01x0w,0xE3F" }, 
    emptyValue = "{}"
)
Output
[1] {}, 1_100
[2] @user-01x0w, 0xE3F

nullValues 속성에 지정한 문자열은 null로 치환된다.

nullValues
@CsvSource(
    value = { "NIL,1_100", "@user-01x0w,0xE3F" },
    nullValues = "NIL"
)
Output
[1] null, 1_100
[2] @user-01x0w, 0xE3F

ignoreLeadingAndTrailingWhitespace = false 설정을 하면 앞뒤 공백을 그대로 둔다.

💡 name 속성은 Display name 커스터마이징 하기 단원에서 설명한다.

@ParameterizedTest(name = "[" + INDEX_PLACEHOLDER + "] ''{0}'', {1}")
@CsvSource(
        value = { "' happy time ',1_100", "@user-01x0w ,0xE3F" },
        ignoreLeadingAndTrailingWhitespace = false
)
Output
[1] ' happy time', 1_100
[2] '@user-01x0w ', 0xE3F

값이 없는 컬럼은 null을 반환한다.

no value
@CsvSource(
    value = { ",1_100", "@user-01x0w,0xE3F" }, 
)
Output
[1] null, 1_100
[2] @user-01x0w, 0xE3F

Java 15 이상에서는 textBlock 속성을 사용해서 가독성을 향상시킬 수 있다. #기호로 시작하는 모든 행은 주석 처리되며, 선행 공백없이 시작되야 한다.

@ParameterizedTest
@CsvSource(delimiter = '|', quoteCharacter = '"', textBlock = """
#-----------------------------
#     NAME     |    SCORE
#-----------------------------
     ninja     |     700
#-----------------------------
    assassin   |     850
#-----------------------------
  "happy time" |    1_100
#-----------------------------
  @user-01x0w  |    0xE3F
#-----------------------------
""")
void CsvSourceWithTextBlockExample(String nickName, int score) {
    //...
}
Output
[1] ninja, 700
[2] assassin, 850
[3] happy time, 1_100
[4] @user-01x0w, 0xE3F

@CsvFileSource

CSV 파일을 가져와서 인수로 지정한다. useHeadersInDisplayName, delimiter 속성은 @CsvSource과 같으며, quoteCharacter 속성은 기본 따옴표가 큰 따옴표인 점이 다르다.

example.csv
#-----------------------------
      NAME     |    SCORE
#-----------------------------
     ninja     |     700
#-----------------------------
    assassin   |     850
#-----------------------------
  "happy time" |    1_100
#-----------------------------
  @user-01x0w  |    0xE3F
#-----------------------------

@ParameterizedTest
@CsvFileSource(resources = "/example.csv", numLinesToSkip = 1, delimiter = '|')
void CsvFileSourceFromClasspathExample(String name, int score) {
    assertNotNull(name);
    assertNotEquals(0, score);
}
@ParameterizedTest
@CsvFileSource(files = "src/test/resources/example.csv", numLinesToSkip = 1, delimiter = '|')
void CsvFileSourceFromFileExample(String name, int score) {
    assertNotNull(name);
    assertNotEquals(0, score);
}
@ParameterizedTest(name = "[{index}] {arguments}")
@CsvFileSource(resources = "/example.csv", useHeadersInDisplayName = true, delimiter = '|')
void CsvFileSourceAndHeadersExample(String name, int score) {
    assertNotNull(name);
    assertNotEquals(0, score);
}

출력 결과는 이미 많이 다뤘으므로 생략한다.

Null and Empty Sources

  • @EmptySource: 인수 값으로 빈문자열을 지정한다.
  • @NullSource: 인수 값으로 null을 지정한다.
  • @NullAndEmptySource : 위의 2개를 합친 어노테이션이다.
import org.junit.platform.commons.util.StringUtils;
@ParameterizedTest
@NullSource
@EmptySource
@ValueSource(strings = { " ", "   ", "\t", "\n" })
void nullEmptyAndBlankStrings(String text) {
    assertTrue(StringUtils.isBlank(text));
}
Output

[2]         nullEmptyAndBlankStrings(String)[2] 0s      passed
[3]         nullEmptyAndBlankStrings(String)[3] 0s      passed
[4]         nullEmptyAndBlankStrings(String)[4] 0s      passed

[6]         nullEmptyAndBlankStrings(String)[6] 0.001s  passed

@EnumSource

Enum 상수를 인수로 지정한다. value 속성에는 enum 클래스를 입력해야 한다. 이 값은 생략 가능하다.

@ParameterizedTest
@EnumSource(ChronoUnit.class)
void enumSourceExample(TemporalUnit unit) {
    assertNotNull(unit);
}
@ParameterizedTest
@EnumSource
void enumSourceWithoutValueExample(ChronoUnit unit) {
    assertNotNull(unit);
}

💡 value 속성 생략시 names 속성에 대해서 자동완성 지원을 받지 못하니, 이점 염두해두길 바란다. Intelli J 2023.1.2에서 확인됨.

names 속성에는 enum 상수의 이름을 문자열로 입력한다. 이 값은 생략 가능하며, 이 경우 모든 상수를 적용한다.

@ParameterizedTest
@EnumSource(value = ChronoUnit.class, names = { "FOREVER", "ERAS" })
void enumSourceIncludeExample(ChronoUnit unit) {
    assertTrue(EnumSet.of(ChronoUnit.FOREVER, ChronoUnit.ERAS).contains(unit));
}

mode 속성에는 enum 클래스에서 어떤 상수를 선택할지에 대해서 모드를 지정한다. INCLUDE, EXCLUDE, MATCH_ALL, MATCH_ANY를 사용할 수 있다.

@ParameterizedTest
@EnumSource(mode = Mode.EXCLUDE, names = { "CENTURIES", "DECADES" })
void EnumSourceExcludeExample(ChronoUnit unit) {
    assertFalse(EnumSet.of(ChronoUnit.CENTURIES, ChronoUnit.DECADES).contains(unit));
}
@ParameterizedTest
@EnumSource(mode = Mode.MATCH_ALL, names = "^.*DAYS$")
void enumSourceRegexExample(ChronoUnit unit) {
    assertTrue(unit.name().endsWith("DAYS"));
}

@MethodSource

하나 이상의 팩토리 메소드를 통해서 인자를 지정한다. 테스트 클래스 내의 팩토리 메소드는 테스트 클래스의 생명주기가 PER_CLASS가 아닌 이상 static 메소드여야 하며, 외부 클래스의 팩토리 메소드는 항상 static 메소드여야 한다.

팩토리 메소드는 Stream<T>을 반환해야 한다. 단일 인수가 필요하다면, 메소드의 인수 타입의 Stream을 반환하면 된다.

@ParameterizedTest
@MethodSource("fruitProvider")
void explicitLocalMethodSourceExample(String parameter) {
    assertNotNull(parameter);
}
static Stream<String> fruitProvider() {
    return Stream.of("cherry", "durian");
}

DoubleStream, IntStream 그리고 LongStream 같은 원시타입도 지원한다.

@ParameterizedTest
@MethodSource("intRange")
void intStreamMethodSourceExample(int parameter) {
    assertNotEquals(9, parameter);
}
static IntStream intRange() {
    return IntStream.rangeClosed(1, 20).skip(10);
}

다중 인수가 필요하다면, arguments(Object ...) 타입의 Stream<Arguments>을 반환하면 된다.

@ParameterizedTest
@MethodSource("workingSchedule")
void multiParamsMethodSourceExample(String name, int workingHours, Set<DayOfWeek> dayOfWeeks) {
    assertNotNull(name);
    assertTrue(workingHours >= 1 && workingHours <= 10);
    assertEquals(2, dayOfWeeks.size());
}
static Stream<Arguments> workingSchedule() {
    return Stream.of(
            arguments("John", 3, EnumSet.of(DayOfWeek.MONDAY, DayOfWeek.WEDNESDAY)),
            arguments("Sam", 9, EnumSet.of(DayOfWeek.TUESDAY, DayOfWeek.THURSDAY))
    );
}

외부 팩토리 메소드를 참조할 때는 아래의 예시를 참고하여 메소드를 참조하도록 한다.

.
└── src
    ├── main
    │   └── ...
    └── test
        ├── java
        │   ├── ...
        │   ├── xyz
        │   │   └── besfir
        │   │       └── StringsProviders.java
        │   └── ExternalMethodSourceDemo.java
        └── resources


import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.MethodSource;
import static org.junit.jupiter.api.Assertions.assertNotNull;
class ExternalMethodSourceDemo {
    @ParameterizedTest
    @MethodSource("some.source.StringsProviders#theSkyIsBlue")
    void externalMethodSourceExample(String parameter) {
        assertNotNull(parameter);
    }
    @ParameterizedTest
    @MethodSource("some.source.StringsProviders$NestingProviders#theFruits")
    void externalMethodSourceWithNestingClassExample(String parameter) {
        assertNotNull(parameter);
    }
}
package xyz.besfir;
import java.util.stream.Stream;
class StringsProviders {
    static Stream<String> theSkyIsBlue() {
        return Stream.of("The", "sky", "is", "blue");
    }
    class NestingProviders {
        static Stream<String> theFruits() {
            return Stream.of("apple", "banana", "cherry", "durian");
        }
    }
}

팩토리 메소드 이름과 @ParameterizedTest 메소드 이름이 같다면 생략이 가능하다.

@ParameterizedTest
@MethodSource
void defaultLocalMethodSourceExample(String argument) {
    assertNotNull(argument);
}
static Stream<String> defaultLocalMethodSourceExample() {
    return Stream.of("apple", "banana");
}

@ArgumentsSource

커스텀 ArgumentsProvider를 지정해 줄 때 사용한다. 위에서 정리한 소스 어노테이션은 내부적으로 @ArgumentsSource와 ArgumentsProvider를 사용해서 기능을 구현하고 있다.

@Target({ ElementType.ANNOTATION_TYPE, ElementType.METHOD })
@Retention(RetentionPolicy.RUNTIME)
@Documented
@API(status = STABLE, since = "5.7")
@ArgumentsSource(ValueArgumentsProvider.class)
public @interface ValueSource {
  ...
}

제공되는 소스 어노테이션 외에 추가적으로 구현하고 싶다면 사용하면 된다.

@RepeatedTest

지정한 횟수만큼 테스트를 반복한다.

@RepeatedTest(10)
void equalsTest() {
    assertEquals(1 + 1, 2);
}

@Nested

테스트를 그룹화 할 때 사용하며, non-static 중첩 클래스에만 사용할 수 있다. Java 16 이전에는 중첩 클래스에 정적 메소드를 선언할 수 없기 때문에 @BeforeAll과 @AfterAll을 사용할 수 없다. 다만, 테스트 인스턴스 생명주기를 PER_CLASS로 하면 사용할 수 있다. Java 16 이후로는 중첩 클래스에 정적 메소드를 선언할 수 있기 때문에 해당되지 않는다.

import org.junit.jupiter.api.Nested;
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertTrue;
class NestedExampleTest {
    @Nested
    class SomeGroup {
        @Nested
        class InnerTest1_1 {
        
            @Test
            void test1() {
                assertTrue(true);
            }
        }
        @Nested
        class InnerTest1_2 {
        
            @Test
            void test2() {
                assertTrue(true);
            }
        }
    }
    @Nested
    class OtherGroup {
        @Nested
        class InnerTestA {
            @Test
            void testA() {
                assertTrue(true);
            }
        }
        @Nested
        class InnerTestB {
        
            @Test
            void testB() {
                assertTrue(true);
            }
        }
    }
}

@Timeout

해당 메소드가 지정한 지속시간을 넘어가면 예외를 발생시키고 즉시 종료하도록 만드는 어노테이션이며, 테스트, 테스트 팩토리, 테스트 템플릿 또는 생명주기 메소드에 선언할 수 있다.
클래스 수준으로 선언해서 여러 메소드에 공통으로 적용할 수도 있으며, @Nested 선언된 중첩 클래스도 영향을 받는다. 단, 클래스 수준으로 설정할 경우 생명주기 메소드에는 적용되지 않는다.

설명만 놓고 보면 assertTimeoutPreemptively()와 쓰임이 겹치는 것 같지만, @Timeout은 특정 시나리오에서 이를 대체하기 위해 사용한다. assertTimeoutPreemptively()는 테스트 메소드와는 다른 스레드에서 실행되기에 spring transaction이 원하는대로 동작하지 않는다. 반면 @Timeout은 기본적으로 테스트 메소드와 같은 스레드에서 동작하기 때문에 앞서 언급한 경우 대체해서 사용된다. 스레드 동작은 threadMode 속성을 통해서 바꿀 수 있다.

  • INFERRED : 기본 값이며, 설정 파라미터 또는 설정 파일의 junit.jupiter.execution.timeout.thread.mode.default property 값을 따른다. 없을시 SAME_THREAD로 동작한다.
  • SAME_THREAD : 테스트 메소드와 같은 스레드에서 실행된다.
  • SEPARATE_THREAD : 테스트 메소드와는 다른 스레드에서 실행된다.

unit 속성으로 단위를 선택할 수 있으며, 기본 단위는 SECONDS이다.

@Test
@Timeout(value = 500, unit = TimeUnit.MILLISECONDS)
void timeoutAnnotationExample() throws InterruptedException {
    Thread.sleep(10_000);
}

@TempDir

임시 디렉터리를 생성하고 정리할 때 사용한다. 파라미터나 필드 수준의 Path나 File타입에 선언할 수 있다.

@Test
void tempDirExample(@TempDir Path source, @TempDir Path target) throws IOException {
    Path sourceFile = source.resolve("test.txt");
    Files.write(sourceFile, "a,b,c".getBytes());
    Path targetFile = Files.copy(sourceFile, target.resolve("test.txt"));
    assertNotEquals(sourceFile, targetFile);
    assertEquals(singletonList("a,b,c"), Files.readAllLines(targetFile));
}

cleanup 속성을 통해 정리 동작을 지정할 수 있으며 DEFAULT, ALWAYS, ON_SUCCESS, NEVER 4가지 모드가 있다. DEFAULT 모드는 설정 파라미터 또는 설정 파일의 junit.jupiter.tempdir.cleanup.mode.default property 값을 따르며, 값이 없을 시 ALWAYS 모드로 동작한다.

Extension Model

선언적 확장 등록

@ExtendWith을 이용해서 하나 이상의 확장을 등록할 수 있다.

@ExtendWith({ DatabaseExtension.class, WebServerExtension.class})
class ExtensionModelTest { }
// or 
@ExtendWith(DatabaseExtension.class)
@ExtendWith(WebServerExtension.class)
class ExtensionModelTest { }

프로그래밍적 확장 등록

@RegisterExtension은 외부에서 확장에 어떤 값을 넣어줘야 할 경우 사용한다. 이때 확장 필드는 private 필드이거나 null이면 안된다.

static 필드에 선언 시에는 @ExtendWith에 등록된 확장보다 이후에 등록된다. 이때, 확장의 클래스 수준 콜백과 메소드 수준 콜백 모두 호출된다.

public class RegisterExtensionTest {
    @RegisterExtension
    static LoggingExtension staticExtension = new LoggingExtension("static version");
    @Test
    public void someTest() { }
}
Output
Type static version In beforeAll : RegisterExtensionTest
Type static version In beforeEach : someTest()

non-static 필드에 선언시에는 @ExtendWith에 지정된 확장보다 나중에 등록되며, 확장의 메소드 수준 콜백만 호출한다.

public class RegisterExtensionTest {
    @RegisterExtension

@TestInstance(TestInstance.Lifecycle.PER_CLASS)

    @RegisterExtension
    LoggingExtension staticExtension = new LoggingExtension("static version");
    @Test
    public void someTest() { }
}

Type non-static version In beforeEach : someTest()

public class LoggingExtension implements BeforeAllCallback, BeforeEachCallback {
    private final String type;
    public LoggingExtension(String type) {
        this.type = type;
    }
    @Override
    public void beforeAll(ExtensionContext extensionContext) throws Exception {
        System.out.printf("Type %s In beforeAll : %s\n", type, extensionContext.getDisplayName());
    }
    @Override
    public void beforeEach(ExtensionContext extensionContext) throws Exception {
        System.out.printf("Type %s In beforeEach : %s\n", type, extensionContext.getDisplayName());
    }
    public String getType() {
        return type;
    }
}

Display name 커스터마이징 하기

@RepeatedTest, @ParameterizedTest와 같이 반복 가능한 테스트는 각 테스트의 식별을 위해서 name 속성으로 표시되는 이름을 사용자 정의할 수 있다. 이때, name 속성은 메세지 포멧 패턴이므로 하나의 작은 따옴표를 표시하기 위해선, 두 번 사용해야한다. 지원되는 placeholder는 아래와 같다.

PlaceholderDescription
{displayName}@DisplayName이 있으면 이를 가져오고, 없으면 메소드 이름을 표시.
{index}1부터 시작하는 현재 인덱스 값을 표시.
{arguments}현재 인수를 콤마로 구분해서 표시한다.
{argumentsWithNames}현재 인수를 인수명과 함께 콤마로 구분해서 표시한다.
(간단히 테스트해봤는데, 위의 것과 다른점을 모르겠다.)
{0}, {1}, ...각 인수 표시.

예제

@DisplayName("Display name of container")
@ParameterizedTest(name = "{index} ==> the rank of ''{0}'' is {1}")
@CsvSource({ "apple, 1", "banana, 2", "'lemon, lime', 3" })
void customDisplayNamesExample(String fruit, int rank) { }
