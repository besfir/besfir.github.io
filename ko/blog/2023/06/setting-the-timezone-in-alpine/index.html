알파인 리눅스 도커 이미지를 이용할때 타임존 설정하기

Besfir · 2023년 6월 6일
목차
  • Alpine linux에서의 타임존 설정
  • Install tzdata package
  • localtime에 타임존 정보 복사
  • 타임존 지정
  • date 명령어로 확인
  • Optimize docker image
  • (Optional) Multi-stage builds 사용시

프로덕션 환경에서는 cron 작업이나 로그 시간 현지화 그리고 DB와 OS 간의 시간 동기화 등의 이유로 직접 타임존을 설정해야하는 경우가 있다. 타임존 설정 방법은 배포판에 따라 차이가 있는데, 이번 글에서는 도커 환경을 구축할때 자주 사용되는 Alpine linux에서의 설정 방법을 알아보고, 도커에서 설정할 시 최적화하는 방법에 대해 알아보겠다.

Alpine linux에서의 타임존 설정

Install tzdata package

알파인 리눅스에는 타임존 설정 관련 패키지가 포함되어 있지 않다. 따라서 tzdata 패키지부터 설치하도록 하곘다.

apk add tzdata

(선택사항) 정상적으로 설치가 됐는지 ls 명령어로 확인을 해봤다.

localtime에 타임존 정보 복사

cp /usr/share/zoneinfo/Asia/Seoul /etc/localtime

타임존 지정

echo "Asia/Seoul" > /etc/timezone

date 명령어로 확인

date
Output
Fri May 12 12:44:46 KST 2023

Optimize docker image

도커에서 timezone을 설정할 때는 위에서 언급한 코드를 약간 변형시킨다. 패키지를 설치할 때 캐시를 사용하지 않기 위해 --no-cache 플래그를 사용하고, 더 이상 필요치 않은 패키지를 삭제하여 용량 최적화를 수행하도록 한다.

FROM node:18.16.0-alpine3.17  

RUN apk add --no-cache tzdata && \
    cp /usr/share/zoneinfo/Asia/Seoul /etc/localtime && \
    echo "Asia/Seoul" > /etc/timezone && \
    apk del tzdata

(Optional) Multi-stage builds 사용시

Multi-stage builds는 docker v17.06부터 지원되는 기능으로, 도커 이미지를 빌드하는 과정에는 필요하지만 최종 결과물에는 필요치 않은 요소들을 쉽게 제외하기 위해서 단계를 나누는 기능이다.

도커 파일이 복잡해지면 결과물에 필요하지 않은 파일을 제거하는 작업이 버거울 수 있다. 이럴 때 multi-stage builds를 사용하면 필요한 자원에만 집중할 수 있다.

FROM node:18.16.0-alpine3.17 AS base

# Builder process
FROM base AS builder

RUN apk add tzdata

# Runner process
FROM base AS runner

COPY --from=builder /usr/share/zoneinfo/Asia/Seoul /etc/localtime
RUN echo "Asia/Seoul" > /etc/timezone
