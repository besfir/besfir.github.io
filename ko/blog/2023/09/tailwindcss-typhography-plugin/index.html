Tailwind CSS Typhography Plugin 사용법 및 커스터마이징

@tailwindcss/typhography는 마크다운에서 랜더링 됐거나 CMS에서 가져온 HTML 등 사용자가 직접 Tailwind CSS 클래스를 지정하지 못하는 HTML 요소에 타이포그래피 기본값을 추가하기 위해 사용되는 플러그인이다. 이 플러그인을 사용하지 않아도 CSS 파일에 직접 오버라이딩 할 수도 있지만, 그렇게 사용할 시 클래스 우선순위 따라 내가 설정한 CSS 클래스가 무시될 수도 있다. 하지만 이 플러그인을 사용하면 개발자가 지정한 스타일을 우선 순위로 사용하기 때문에 편리하다는 장점이 있다.

설치

pnpm i -D @tailwindcss/typography
tailwind.config.js
/** @type {import('tailwindcss').Config} */
module.exports = {
  theme: {
    // ...
  },
  plugins: [
    require('@tailwindcss/typography'),
    // ...
  ],
}

기본 사용법

플러그인을 추가했다고 해서 즉시 글로벌하게 적용되는 것은 아니며, prose 클래스를 추가해준 요소에 한해서 적용된다. 이해를 돕기 위해 Tailwind CSS에서 제공하는 데모의 일부를 가져왔다.

<article class="prose">
  <h1>Garlic bread with cheese: What the science tells us</h1>
  <p>
    For years parents have espoused the health benefits of eating garlic bread with cheese to their
    children, with the food earning such an iconic status in our culture that kids will often dress
    up as warm, cheesy loaf for Halloween.
  </p>
  <p>
    But a recent study shows that the celebrated appetizer may be linked to a series of rabies cases
    springing up around the country.
  </p>
  <!-- ... -->
</article>
  • 1번 행에 prose 클래스를 추가했기 때문에 해당 요소에는 기본 타이포그래피 스타일이 적용된다.

테마 선택하기

@tailwindcss/typography 플러그인은 본문에 사용하기 좋은 5가지 회색 계열 색상을 제어자 클래스(modifier class)로 준비해 놓았다. 이 프리셋은 body, headings, links 등 모든 요소 제어자(element modifier)의 색상을 바꾸므로 테마라고 할 수 있다. 따리서 이후에는 테마라고 부르도록 하겠다.

ClassGray scale
prose-gray (default)Gray
prose-slateSlate
prose-zincZinc
prose-neutralNeutral
prose-stoneStone

사용하기 위해서는 원하는 요소에 상기 클래스를 추가하면 된다. 주의할 점으로 제어자 클래스는 prose 클래스와 함께 사용하도록 설계되었기 때문에 항상 prose 클래스와 함께 사용해야 한다.

<article class="prose prose-stone">
  {{ markdown }}
</article>

각 테마마다 요소에 적용된 색상이 궁금하다면 GitHub code에서 확인 가능하다.

크기 제어자 선택하기

크기 제어자(size modifier)를 이용하면 타이포그래피의 전체 크기를 조정할 수 있다.

ClassBody font size
prose-sm0.875rem(14px)
prose-base1rem(16px)
prose-lg1.125rem(18px)
prose-xl1.25rem(20px)
prose-2xl1.5rem(24px)

가령, 스마트폰과 테블릿 기기에서는 16px 폰트를 사용하고 랩탑 사이즈 이상의 기기에서는 18px 폰트를 사용하고 싶다면 아래와 같이 사용할 수 있다.

<article class="prose lg:prose-lg">
  {{ markdown }}
</article>
  • 16px 폰트는 기본값이므로 생략했다.
  • 반응형 제어자(responsive modifier) lg를 사용해서 뷰포트 크기 1024px 이상의 기기에서는 prose-lg를 사용하도록 했다.
    • prose-lg는 body font size가 18px이다.

다크 모드 적용

각 기본 색상 테마에는 다크 모드도 포함되어 있다. 따라서 별다른 설정을 해줄 것 없이, 클래스를 추가해서 사용하면 된다.

<article class="prose dark:prose-invert">
  {{ markdown }}
</article>
  • dark variant를 이용해서 다크 모드에서 사용할 prose-invert 클래스를 추가했다.

요소 제어자

요소 제어자(element modifier)를 사용해서 HTML 개별 요소에 대해 스타일을 커스터마이징 할 수 있다. 이를 통해 브랜드 스타일에 어울리는 링크 색상, 이미지 테두리, heading 태그 모양 변경 등의 작업을 쉽게 진행할 수 있다.

<article class="prose prose-img:rounded-xl prose-headings:underline prose-a:text-blue-600">
  {{ markdown }}
</article>
  • prose-img:rounded-xl : 이미지 태그에 border-radius을 0.75rem 적용한다.
  • prose-headings:underline : h1, h2...와 같은 heading 태그에 밑줄을 적용한다.
  • prose-a:text-blue-600 : a태그에 #2563eb 색상을 적용한다.

사용 가능한 요소 제어자 목록은 다음과 같다.

ModifierTarget
prose-headings:{utility}h1, h2, h3, h4, th
prose-lead:{utility}[class~="lead"]
prose-h1:{utility}h1
prose-h2:{utility}h2
prose-h3:{utility}h3
prose-h4:{utility}h4
prose-p:{utility}p
prose-a:{utility}a
prose-blockquote:{utility}blockquote
prose-figure:{utility}figure
prose-figcaption:{utility}figcaption
prose-strong:{utility}strong
prose-em:{utility}em
prose-code:{utility}code
prose-pre:{utility}pre
prose-ol:{utility}ol
prose-ul:{utility}ul
prose-li:{utility}li
prose-table:{utility}table
prose-thead:{utility}thead
prose-tr:{utility}tr
prose-th:{utility}th
prose-td:{utility}td
prose-img:{utility}img
prose-video:{utility}video
prose-hr:{utility}hr

전체 너비 사용하기

prose 클래스는 가독성을 위해 각 65ch로 최대 너비가 제한된다. 전체 너비를 사용하고 싶다면 max-w-none 클래스를 추가하면 된다.

<div class="grid grid-cols-4">
  <div class="col-span-1">
    <!-- ... -->
  </div>
  <div class="col-span-3">
    <article class="prose max-w-none">
      {{ markdown }}
    </article>
  </div>
</div>

고급 사용법

prose 클래스가 적용된 요소 안에서 타이포그래피 스타일의 영향을 받지 않고 싶다거나, 기본 제공되는 테마 외에 커스텀 테마를 꾸미고 싶은 경우 그리고 CSS 전체를 수정하고 싶은 경우에는 어떻게 해야할지 정리해보자.

타이포그래피 스타일 제외하기

not-prose라는 클래스를 사용하면 prose 클래스가 추가된 요소안에서 타이포그래피 스타일의 영향을 받지 않는다.

<article class="prose">
  <h1>Some Heading</h1>
  <p>...</p>

  <div class="not-prose">
    <!-- 타이포그래피 스타일을 적용하고 싶지 않은 요소들 -->
  </div>

  <p>...</p>
  <!-- ... -->
</article>

not-prose 내부에 prose를 쓴다고 해도 타이포그래피 스타일을 사용할 순 없다는 점을 염두해 두도록 한다.

CSS 커스터마이징 하기

이 부분이 이 글을 쓴 목적이다. 타이포그래피 스타일을 커스터마이징 하기 위해선 typography 색션에 미리 정의된 키에 맞춰 값을 입력해야 한다.

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [],
  theme: {
    extend: {
      typography: theme => ({
        invert: {
          css: {},
        },
        DEFAULT: {
          css: {},
        },
        sm: {
          css: {},
        },
        // ...
      }),
    },
  },
  plugins: [require("@tailwindcss/typography")],
}

미리 준비된 키값은 다음과 같다.

  • 크기 : sm, base, lg, xl, 2xl
  • 테마 : slate, gray, zinc, neutral, stone, red, orange, amber, yellow, lime, green, emerald, teal, cyan, sky, blue, indigo, violet, purple, fuchsia, pink, rose,
  • 다크 모드 : invert

상기 키값을 사용하려면 prose-[key]를 입력하면 된다. 각 키마다 스타일이 적용됐는지는 Github code에서 확인할 수 있으니 참조바란다.

새로운 스타일 추가하기

정의된 키값 외에도 본인이 새로이 키값을 정의해서 사용할 수 있다. 가령 md라는 키값을 추가한다고 가정해보자.

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [],
  theme: {
    extend: {
      typography: theme => ({
        md: {
          css: {
            a: {
              color: theme('colors.red.500'),
            },
          },
        },
      }),
    },
  },
  plugins: [require("@tailwindcss/typography")],
}

<article class="prose lg:prose-md">
  {{ markdown }}
</article>
  • 뷰포트 사이즈가 1024px을 넘어가면 prose-md가 적용되어 <a> 태그가 붉은 색이 된다.

해당 데모를 tailwind playground에서 작성했으니 참조바란다.
