Nuxt i18n 루트 경로 상에서 기본 로케일로 전환할 수 없는 문제 트러블 슈팅

배경

Nuxt-i18n v8.0.0-beta.10으로 작업을 하던 중, switchLocalePath() 함수를 이용해서 루트 경로로 이동하지 못하는 현상을 발견했다. 다른 경로 간 라우팅은 문제가 없는데, 루트 경로에서 로케일 전환을 통한 라우팅만 문제가 있었다.

문제 해결 과정

문제 범위 좁히기

루트 경로에서만 문제가 발생하는 상황으로 유추해 볼 때, 브라우저 언어 감지에 관한 설정 쪽에서 문제가 생긴 것으로 예상된다. 해당 부분에 루트 경로가 명시적으로 관련된 부분이 있기 때문이다. 예상했던 대로 가장 의심스러운 alwaysRedirect: true 옵션을 제거해 보니 로케일 전환이 정상적으로 작동했다.

nuxt.config.ts
i18n: {
  baseUrl: config.baseUrl,
  strategy: "prefix_except_default",
  defaultLocale: "en",
  detectBrowserLanguage: {
    useCookie: true,
    cookieKey: "i18n_redirected",
    redirectOn: "root",
    alwaysRedirect: true,
    fallbackLocale: "en",
  },
},

임시 해결책

원인은 알겠는데 도저히 해결 방법이 떠오르지 않아서 한동안 prefix 전략으로 사용했다. 이 전략을 사용하면 /en <==> /ko 전환만 하면 되기 때문에 위의 설정이 영향을 주지 못해 대안을 찾기전까지 임시로 적용하기로 했다.

관련 정보 검색

작동은 했지만 원하는 방법이 아니었고 prefix except default 전략을 사용하는 것이 더 자연스러워 보였기에 어떻게 해서든 이를 적용하고 싶었다. 따라서 검색으로 문제를 해결해야 했는데, 이 문제처럼 보편적이지 않은 문제가 생겼을 때에는 검색 키워드를 정하는게 정말 어렵다.

결국 검색 키워드를 정하지 못하고 무작정 Nuxt-i18n Github에 들어가서 비슷한 이슈를 찾기 시작했는데, 다행히 오래 걸리지 않고 비슷한 Github issue를 찾을 수 있었다.

해결

BobbieGoede의 코드로 문제를 해결할 수 있었다. 다만, 같은 기능을 한다면 더 간단한 코드를 선호하기 때문에 nuxt-link를 확장하지 않고 클릭 이벤트를 직접 처리하도록 했다.

Code

현재 사용중인 코드 일부를 가져왔다.

LangSwitcher.vue
<script setup lang="ts">
import { LocaleObject } from "vue-i18n-routing"

const { locale, locales } = useI18n()
const switchLocalePath = useSwitchLocalePath()

const availableLocales = computed(() => {
  return (locales.value as LocaleObject[]).filter(i => i.code !== locale.value)
})

const router = useRouter()
const cookie = useCookie("i18n_redirected")
const onClickLocaleSwitch = (code: string) => {
  cookie.value = code
  router.push(switchLocalePath(code))
}
</script>

<template>
  <nuxt-link
    v-for="locale in availableLocales"
    :key="locale.code"
    :to="switchLocalePath(locale.code)"
    @click.prevent="onClickLocaleSwitch(locale.code)"
  >
    <icon name="mdi:translate" size="20" class="mr-1" />
    {{ locale.name }}
  </nuxt-link>
</template>
nuxt.config.ts
modules: ['@nuxtjs/i18n'],
i18n: {
  baseUrl: config.baseUrl,
  langDir: "./helpers/locales",
  locales: [
    { code: "ko", iso: "ko-KR", name: "한국어", file: "ko.json" },
    { code: "en", iso: "en-US", name: "English", file: "en.json" },
  ],
  strategy: "prefix_except_default",
  defaultLocale: "en",
  detectBrowserLanguage: {
  useCookie: true,
    cookieKey: "i18n_redirected",
    redirectOn: "root",
    alwaysRedirect: true,
    fallbackLocale: "en",
  },
},
