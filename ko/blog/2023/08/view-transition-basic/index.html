Chrome 111 버전부터 흥미로운 기능인 "View Transition API"가 추가되었다. 이번 글에서는 View Transition API에 대해 알아보도록 하겠다.

View Transition API는 무엇인가?

View Transition API는 서로 다른 DOM 상태 간에 애니메이션 트랜지션을 간편하게 생성할 수 있는 동시에 DOM 콘텐츠를 한 번에 업데이트하는 기능을 제공한다. 어디에 사용할 수 있는지 몇가지 예시를 확인해보자.

사용자 상호작용으로 요소의 순서가 바뀌는 경우, 바뀐 요소의 식별이 어려울수도 있다. 이때, 바뀐 순서에 맞춰 정렬 애니메이션이 발생한다면 사용자로 하여금 순서를 변경했다는 것을 쉽게 인지시킬 수 있다. 위 데모 영상은 draggable 요소에 View Transition API를 적용해서 재정렬을 쉽게 인지할 수 있도록 했다.

백엔드 프레임워크를 활용한 SSR은 MPA로서, SPA와 비교하면 페이지 전환 애니메이션 효과를 구현하는데 훨씬 공수가 많이 든다. 하지만 View Transition API를 활용하면 페이지 전환 효과를 비교적 쉽게 구현할 수 있다. 위 데모 영상은 Express + Handlebars에 View Transition API를 적용해서 페이지 전환 효과를 구현했으며, 페이지가 전환되어도 동영상은 끊기지 않고 재생된다.

실습해보자

💡 아직 실험적 기능 단계이기 때문에 크로미움 엔진 기반 브라우저인 크롬, 엣지, 오페라에서만 동작한다.

실습을 위해서 버튼을 클릭하면 도형의 형태가 토글되는 간단한 예제를 만들었다. 이 예제를 기반으로 코드를 추가하며 사용법을 간단히 알아보겠다.

View Transition 적용

기본 사용 방법은 정말 간단하다. 그저 startViewTransition() 안에서 DOM을 업데이트 하면 바로 트랜지션이 적용된다.

const shape = document.getElementById('shape')
shape.addEventListener('click', () => document.startViewTransition(handleClick))
function handleClick() {
  shape.classList.toggle('active')
}

기존에는 클릭시 직접 updateDOM()을 호출하던 것을 View Transition API를 통해 업데이트 하도록 변경했다. 버튼을 클릭 해보면, 기본 트랜지션 효과인 cross-fade 효과가 적용되는 것을 볼 수 있다.

어떻게 동작하는걸까?

View Transition API는 동작 과정을 간단히 아래의 단계로 표현할 수 있다.

  1. startViewTransition()이 호출되면 API는 현재 페이지의 스크린샷을 찍는다.
  2. startViewTransition()에 넘긴 콜백이 호출되어 DOM이 변경된다.
  3. 변경된 페이지의 새로운 상태를 live representation으로 캡쳐한다.
  4. 다음의 구조로 pseudo-element 트리를 구성한다.
::view-transition
└─ ::view-transition-group(root)
   └─ ::view-transition-image-pair(root)
      ├─ ::view-transition-old(root)
      └─ ::view-transition-new(root)
  • ::view-transition은 모든 View Transition pseudo-element 오버레이를 포함하는 최상위 pseudo-element이다.
    • CSS 속성 view-transition-name을 추가하면, 해당 이름의 ::view-transition-group(some-name) 오버레이가 생성된다.
  • ::view-transition-old(root)은 1에서 작업한 이전 페이지의 스크린샷이며, ::view-transition-new(root)은 3에서 캡쳐한 live representation이다.
    • 이 두가지 모두 <img> 또는 <video>와 비슷한 콘텐츠로 랜더링되므로 object-fit이나 object-position과 같은 속성으로 스타일을 지정할 수 있다.
  1. ::view-transition-old(root)과 ::view-transition-new(root)에는 기본 효과인 cross-fade가 적용되서 부드럽게 전환된다.

커스텀 트랜지션

이번에는 기본 트랜지션을 원하는 다른 효과로 바꿔보자. 업데이트 전의 요소들이 왼쪽으로 fade out 되면서 사라지고, 업데이트 후의 요소들이 오른쪽으로부터 fade in 되면서 나타나는 효과를 적용해보겠다.

@keyframes fade-in {
  from { opacity: 0; }
}
@keyframes fade-out {
  to { opacity: 0; }
}
@keyframes slide-from-right {
  from { transform: translateX(50px); }
}
@keyframes slide-to-left {
  to { transform: translateX(-50px); }
}
::view-transition-old(root) {
  animation: 190ms cubic-bezier(0.4, 0, 1, 1) both fade-out,
    500ms cubic-bezier(0.4, 0, 0.2, 1) both slide-to-left;
}
::view-transition-new(root) {
  animation: 310ms cubic-bezier(0, 0, 0.2, 1) 190ms both fade-in,
    500ms cubic-bezier(0.4, 0, 0.2, 1) both slide-from-right;
}

이전에는 티가 나지 않았지만, 결과를 확인해보니 헤더에도 트랜지션 효과가 적용된 것을 알 수 있다. 웹 페이지 이동을 한다고 생각해보면 헤더에도 트랜지션 효과가 적용되는 것은 다소 어색하다.

요소별 각기 다른 효과

이전의 예제에서 어색한 부분을 지적했는데, 이를 바탕으로 두 가지 요구사항을 만들어서 그에 맞춰 수정해보겠다.

  • DOM 업데이트 전/후 헤더는 변화가 없기 때문에 트랜지션을 적용하고 싶지 않다.
  • 도형의 크기가 변화함에 따라 버튼이 자연스럽게 이동하도록 만들고 싶다.
header{
  view-transition-name: header;
}
#activator{
  view-transition-name: activator;
}
@keyframes fade-in {
  from { opacity: 0; }
}
@keyframes fade-out {
  to { opacity: 0; }
}
@keyframes slide-from-right {
  from { transform: translateX(50px); }
}
@keyframes slide-to-left {
  to { transform: translateX(-50px); }
}
::view-transition-old(root) {
  animation: 190ms cubic-bezier(0.4, 0, 1, 1) both fade-out,
  500ms cubic-bezier(0.4, 0, 0.2, 1) both slide-to-left;
}
::view-transition-new(root) {
  animation: 310ms cubic-bezier(0, 0, 0.2, 1) 190ms both fade-in,
  500ms cubic-bezier(0.4, 0, 0.2, 1) both slide-from-right;
}

어떻게 작동하는걸까?

view-transition-name 값을 주게 되면, DOM 업데이트 전/후 이 값이 같은 요소로 부드럽게 전환된다.

  • 헤더는 DOM 업데이트 전/후 전화가 없기 때문에 부드럽게 전환될만한 효과가 없으므로 그대로 유지된다.
  • 버튼은 DOM 업데이트 후에 도형의 크기에 따라 위치가 위 또는 아래로 바뀐다. 따라서 부드러운 전환을 위해 이동하게 된다.

앞서 언급한 것처럼 view-transition-name 값을 지정하면 그 값에 해당하는 ::view-transition-group(some-name)이 생성된다. 따라서 별도의 오버레이로 작동되기 때문에 각기 다른 효과를 구현할 수 있게 되는 것이다.

::view-transition
├─ ::view-transition-group(root)
│  └─ ::view-transition-image-pair(root)
│     ├─ ::view-transition-old(root)
│     └─ ::view-transition-new(root)
├─ ::view-transition-group(header)
│  └─ ::view-transition-image-pair(header)
│     ├─ ::view-transition-old(header)
│     └─ ::view-transition-new(header)
└─ ::view-transition-group(activator)
   └─ ::view-transition-image-pair(activator)
      ├─ ::view-transition-old(activator)
      └─ ::view-transition-new(activator)

view-transition-name 값은 고유해야 하며, 만약 한 폐이지 안에 중복되는 값이 있다면 트랜지션 효과는 무시된다. 도형에도 고유한 view-transition-name 값을 줘서 어떻게 변하는지 확인해보는 것도 좋다.

여기까지 View Transition API에 대해서 간단히 알아봤다. 다음 글에서는 Express + Handlebars로 랜더링한 MPA에 View Transition API를 적용해서 페이지 전환 효과를 구현해보겠다.
