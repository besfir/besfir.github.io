이번 글에서는 데이터 계층 슬라이스 테스트 어노테이션 중 JPA 테스트를 위한 @DataJpaTest에 대해 살펴보고, 이와 관련된 @AutoConfigureTestDatabase에 대해서도 알아보겠다.

@DataJpaTest

JPA 컴포넌트에 집중해서 테스트를 수행하기 위해 사용하며, 전체 자동 구성이 비활성화되고 대신 JPA 테스트와 관련된 구성만 적용된다.

이해해 두면 좋을 점은, @DataJpaTest 내부에 @Transactional이 존재하기 때문에 테스트가 정상적으로 실행되더라도 데이터가 롤백 된다는 것이다. 또한 @DataJpaTest 내부에 @AutoConfigureTestDatabase이 존재하기 때문에 datasource 관련 설정을 무시하고 임베디드 데이터베이스를 사용한다.

showSql

spring.jpa.show-sql과 같은 역할을 하는 속성으로, SQL 로그 출력 여부를 지정한다. 기본값은 true이다.

bootstrapMode

레포지토리 bean의 초기화 시점을 의미하는 부트스트랩 모드를 지정한다. 아래 예제에서 bootstrapMode만 바꿔서 로그를 확인해 보겠다.

@DataJpaTest(bootstrapMode = BootstrapMode.DEFAULT)
class TestUsingDataJpaTest {
    @Autowired
    private MemberRepository memberRepository;
    @Test
    void createMemberTest() {
        Member member = new Member();
        member.setAge(10);
        member.setUsername("foo");
        memberRepository.save(member);
    }
}

properties

이전 글에서 다루었던 @TestPropertySource 의 속성과 동일하다. 테스트를 실행하기 전에 스프링 환경에 추가해야 하는 key=value 형식의 속성이다.

useDefaultFilters

@SpringBootApplication에 사용되는 필터를 테스트에도 적용할지 여부를 지정하는 속성이다. 기본값은 true이며, 어떠한 bean도 포함하지 않는다.

includeFilters

사용시 특정 조건을 만족하는 컴포넌트 클래스들만 bean으로 등록할 수 있다. 즉, 특정 타입이나 어노테이션 등을 기준으로, bean으로 등록할 컴포넌트 클래스를 필터링할 수 있다.

@DataJpaTest(
  includeFilters = @ComponentScan.Filter(
    type = FilterType.ANNOTATION,
    classes = {Controller.class, Service.class}
  )
)
class DataLayerTest { ... }

excludeFilters

includeFilters 속성과 반대로, bean으로 등록하지 않을 컴포넌트 클래스를 필터링하기 위한 속성이다.

@DataJpaTest(
  excludeFilters = @ComponentScan.Filter(
    type = FilterType.ASSIGNABLE_TYPE,
    classes = {GenericRepository.class}
  )
)
class DataLayerTest { ... }

excludeAutoConfiguration

특정 자동 구성 클래스들을 제외하고 싶을 때 사용하는 속성입니다. 이래 예제의 경우, 자동 구성에서 Flyway를 제외한다.

@DataJpaTest(excludeAutoConfiguration = FlywayAutoConfiguration.class)
class DataLayerTest { ... }

@AutoConfigureTestDatabase

애플리케이션 정의 또는 자동 구성된 datasource을 대신해서 사용할 테스트 데이터베이스를 구성하기 위해 사용한다. 여러 datasource bean이 있는 경우 @Primary datasource에만 적용된다. @SpringBootTest와도 조합이 가능하다.

replace

테스트용 임베디드 데이터베이스로 교체할 조건을 지정한다.

  • ANY

기본값이며, Datasource bean이 자동 구성되었든 수동으로 정의되었든 관계없이 datasource bean을 교체한다.

  • AUTO_CONFIGURED

특별히 datasource 관련 설정을 하지 않아서, datasource가 자동 구성된 경우에만 교체한다.

  • NONE

어떠한 경우에도 임베디드 데이터베이스를 사용하지 않는다.

connection

테스트용 데이터베이스로 어떤 임베디드 데이터베이스를 사용할지 지정하는 속성이다.

  • NONE

기본값이며, classpath에 접근해서 탐지를 시도한다.

  • H2

H2 데이터베이스로 지정한다.

  • DERBY

DERBY 데이터베이스로 지정한다.

  • HSQLDB

HSQLDB 데이터베이스로 지정한다.
