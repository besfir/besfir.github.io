@ParameterizedTest

It is used to repeat the test several times using different arguments. Used in conjunction with the source annotation that specifies the value of the argument to be repeated. There are multiple source annotations, but knowing @ValueSource, @CsvSource is big enough.

@ValueSource

Specifies a single array of literal values as an argument. The supported types are short, byte, char, int, long, float, double, boolean, String and Class

@ParameterizedTest
@ValueSource(ints = {1, 2, 3, 4, 5})
void parameterizedTestExample(int candidate) {
    assertTrue(candidate < 10);
}

@CsvSource

Specifies comma-separated values as arguments. Each argument represents a CSV record.

@ParameterizedTest
@CsvSource({"ninja,700", "assassin,850", "'happy time',1_100", "@user-01x0w,0xE3F" })
void csvSourceExample(String name, int score) { }
Output
[1] ninja, 700
[2] assassin, 850
[3] happy time, 1_100
[4] @user-01x0w, 0xE3F

💡 From here on, I will omit the unnecessary parts so that I can focus on the changed attribute values of @CsvSource.

The useHeadersInDisplayName property allows the first record to be optionally used as a CSV header. If set to true, the first record is recognized as a header, skipped, and displayed as Display name.

useHeadersInDisplayName
@CsvSource(
    value = {"NAME,SCORE", "'happy time',1_100", "@user-01x0w,0xE3F"},
    useHeadersInDisplayName = true
)
Output
[1] NAME = happy time, SCORE = 1_100
[2] NAME = @user-01x0w, SCORE = 0xE3F

The default delimiter is a comma (','). Char type delimiter can be specified through the delimiter attribute, and String type delimiter can be specified through the delimiterString attribute. Using the two properties together results in PreconditionViolationException.

delimiter
@CsvSource(
    value = { "'happy time':1_100", "@user-01x0w:0xE3F" }, 
    delimiter = ':'
)
Output
[1] happy time, 1_100
[2] @user-01x0w, 0xE3F

Use the default quotation marks (') as quotation marks. An empty string enclosed in quotation marks returns an empty string unless you set a separate value for the emptyValue attribute.

quoteCharacter
@CsvSource(
    value = { "\"happy time\",1_100", "@user-01x0w,0xE3F" }, 
    quoteCharacter = '\"'
)
Output
[1] happy time, 1_100
[2] @user-01x0w, 0xE3F
empty string
@CsvSource({"'',1_100", "@user-01x0w,0xE3F" })
Output
[1] , 1_100
[2] @user-01x0w, 0xE3F
emptyValue
@CsvSource(
    value = { "'',1_100", "@user-01x0w,0xE3F" }, 
    emptyValue = "{}"
)
Output
[1] {}, 1_100
[2] @user-01x0w, 0xE3F

The string specified in the nullValues property is replaced with null.

nullValues
@CsvSource(
    value = { "NIL,1_100", "@user-01x0w,0xE3F" },
    nullValues = "NIL"
)
Output
[1] null, 1_100
[2] @user-01x0w, 0xE3F

If ignoreLeadingAndTrailingWhitespace = false is set, the front and back spaces are left intact.

💡 The name attribute will be described in the Customizing display name section.

@ParameterizedTest(name = "[" + INDEX_PLACEHOLDER + "] ''{0}'', {1}")
@CsvSource(
        value = { "' happy time ',1_100", "@user-01x0w ,0xE3F" },
        ignoreLeadingAndTrailingWhitespace = false
)
Output
[1] ' happy time', 1_100
[2] '@user-01x0w ', 0xE3F

A column with no value returns null.

no value
@CsvSource(
    value = { ",1_100", "@user-01x0w,0xE3F" }, 
)
Output
[1] null, 1_100
[2] @user-01x0w, 0xE3F

In Java 15 and later, textBlock properties can be used to improve readability. All rows beginning with the # symbol are treated as annotations and must begin without leading spaces.

@ParameterizedTest
@CsvSource(delimiter = '|', quoteCharacter = '"', textBlock = """
#-----------------------------
#     NAME     |    SCORE
#-----------------------------
     ninja     |     700
#-----------------------------
    assassin   |     850
#-----------------------------
  "happy time" |    1_100
#-----------------------------
  @user-01x0w  |    0xE3F
#-----------------------------
""")
void CsvSourceWithTextBlockExample(String nickName, int score) {
    //...
}
Output
[1] ninja, 700
[2] assassin, 850
[3] happy time, 1_100
[4] @user-01x0w, 0xE3F

@CsvFileSource

Gets the CSV file and specifies it as an argument. The useHeadersInDisplayName, delimiter attribute is the same as @CsvSource, and the quoteCharacter attribute is different in that the default quotation marks are double quotes.

example.csv
#-----------------------------
      NAME     |    SCORE
#-----------------------------
     ninja     |     700
#-----------------------------
    assassin   |     850
#-----------------------------
  "happy time" |    1_100
#-----------------------------
  @user-01x0w  |    0xE3F
#-----------------------------
@ParameterizedTest
@CsvFileSource(resources = "/example.csv", numLinesToSkip = 1, delimiter = '|')
void CsvFileSourceFromClasspathExample(String name, int score) {
    assertNotNull(name);
    assertNotEquals(0, score);
}

@ParameterizedTest
@CsvFileSource(files = "src/test/resources/example.csv", numLinesToSkip = 1, delimiter = '|')
void CsvFileSourceFromFileExample(String name, int score) {
    assertNotNull(name);
    assertNotEquals(0, score);
}

@ParameterizedTest(name = "[{index}] {arguments}")
@CsvFileSource(resources = "/example.csv", useHeadersInDisplayName = true, delimiter = '|')
void CsvFileSourceAndHeadersExample(String name, int score) {
    assertNotNull(name);
    assertNotEquals(0, score);
}

Null and Empty Sources

  • @EmptySource : Specifies an empty string as an argument value.
  • @NullSource : Specifies null as the argument value.
  • @NullAndEmptySource : It is an annotation that combines the two above.
import org.junit.platform.commons.util.StringUtils;

@ParameterizedTest
@NullSource
@EmptySource
@ValueSource(strings = { " ", "   ", "\t", "\n" })
void nullEmptyAndBlankStrings(String text) {
    assertTrue(StringUtils.isBlank(text));
}
Output
[1] null    nullEmptyAndBlankStrings(String)[1] 0.012s  passed
[2]         nullEmptyAndBlankStrings(String)[2] 0s      passed
[3]         nullEmptyAndBlankStrings(String)[3] 0s      passed
[4]         nullEmptyAndBlankStrings(String)[4] 0s      passed
[5]         nullEmptyAndBlankStrings(String)[5] 0s      passed
[6]         nullEmptyAndBlankStrings(String)[6] 0.001s  passed

@EnumSource

Specifies the Enum constant as an argument. For the value attribute, you must enter an enum class. This value is optional.

@ParameterizedTest
@EnumSource(ChronoUnit.class)
void enumSourceExample(TemporalUnit unit) {
    assertNotNull(unit);
}

@ParameterizedTest
@EnumSource
void enumSourceWithoutValueExample(ChronoUnit unit) {
    assertNotNull(unit);
}

💡 If you omit the value attribute, you will not receive automatic completion support for the names attribute, so please keep this in mind. Checked in Intelli J 2023.1.2.

In the names attribute, enter the name of the enum constant as a string. This value is optional, in which case all constants are applied.

@ParameterizedTest
@EnumSource(value = ChronoUnit.class, names = { "FOREVER", "ERAS" })
void enumSourceIncludeExample(ChronoUnit unit) {
    assertTrue(EnumSet.of(ChronoUnit.FOREVER, ChronoUnit.ERAS).contains(unit));
}

The mode attribute specifies the mode for which constant to select in the enum class. INCLUDE, EXCLUDE, MATCH_ALL and MATCH_ANY can be used.

@ParameterizedTest
@EnumSource(mode = Mode.EXCLUDE, names = { "CENTURIES", "DECADES" })
void EnumSourceExcludeExample(ChronoUnit unit) {
    assertFalse(EnumSet.of(ChronoUnit.CENTURIES, ChronoUnit.DECADES).contains(unit));
}

@ParameterizedTest
@EnumSource(mode = Mode.MATCH_ALL, names = "^.*DAYS$")
void enumSourceRegexExample(ChronoUnit unit) {
    assertTrue(unit.name().endsWith("DAYS"));
}

@MethodSource

Specify arguments through one or more factory methods. The factory method within the test class should be a static method unless the life cycle of the test class is PER_CLASS, and the factory method of the external class should always be a static method.

The factory method should return Stream<T>. If a single argument is required, just return the Stream of the method's argument type.

@ParameterizedTest
@MethodSource("fruitProvider")
void explicitLocalMethodSourceExample(String parameter) {
    assertNotNull(parameter);
}

static Stream<String> fruitProvider() {
    return Stream.of("cherry", "durian");
}

It also supports stream for primitive types such as DoubleStream, IntStream, and LongStream.

@ParameterizedTest
@MethodSource("intRange")
void intStreamMethodSourceExample(int parameter) {
    assertNotEquals(9, parameter);
}

static IntStream intRange() {
    return IntStream.rangeClosed(1, 20).skip(10);
}

If multiple arguments are required, return Stream<Arguments> of type arguments(Object...).

@ParameterizedTest
@MethodSource("workingSchedule")
void multiParamsMethodSourceExample(String name, int workingHours, Set<DayOfWeek> dayOfWeeks) {
    assertNotNull(name);
    assertTrue(workingHours >= 1 && workingHours <= 10);
    assertEquals(2, dayOfWeeks.size());
}

static Stream<Arguments> workingSchedule() {
    return Stream.of(
            arguments("John", 3, EnumSet.of(DayOfWeek.MONDAY, DayOfWeek.WEDNESDAY)),
            arguments("Sam", 9, EnumSet.of(DayOfWeek.TUESDAY, DayOfWeek.THURSDAY))
    );
}

Refer to the example below when referring to the external factory method.

.
└── src
    ├── main
    │   └── ...
    └── test
        ├── java
        │   ├── ...
        │   ├── xyz
        │   │   └── besfir
        │   │       └── StringsProviders.java
        │   └── ExternalMethodSourceDemo.java
        └── resources
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.MethodSource;

import static org.junit.jupiter.api.Assertions.assertNotNull;

class ExternalMethodSourceDemo {

    @ParameterizedTest
    @MethodSource("some.source.StringsProviders#theSkyIsBlue")
    void externalMethodSourceExample(String parameter) {
        assertNotNull(parameter);
    }
    @ParameterizedTest
    @MethodSource("some.source.StringsProviders$NestingProviders#theFruits")
    void externalMethodSourceWithNestingClassExample(String parameter) {
        assertNotNull(parameter);
    }
}
package xyz.besfir;

import java.util.stream.Stream;

class StringsProviders {

    static Stream<String> theSkyIsBlue() {
        return Stream.of("The", "sky", "is", "blue");
    }

    class NestingProviders {

        static Stream<String> theFruits() {
            return Stream.of("apple", "banana", "cherry", "durian");
        }
    }
}

If the factory method name and the @ParameterizedTest annotated method name are the same, it can be omitted.

@ParameterizedTest
@MethodSource
void defaultLocalMethodSourceExample(String argument) {
    assertNotNull(argument);
}

static Stream<String> defaultLocalMethodSourceExample() {
    return Stream.of("apple", "banana");
}

@ArgumentsSource

Used to specify a custom ArgumentsProvider. The source annotations summarized above are internally implementing functions using @ArgumentsSource and ArgumentsProvider.

@Target({ ElementType.ANNOTATION_TYPE, ElementType.METHOD })
@Retention(RetentionPolicy.RUNTIME)
@Documented
@API(status = STABLE, since = "5.7")
@ArgumentsSource(ValueArgumentsProvider.class)
public @interface ValueSource {
  ...
}

You can use it if you want to implement it additionally in addition to the source annotation provided.

@RepeatedTest

Repeat the test as many times as you specify.

@RepeatedTest(10)
void equalsTest() {
    assertEquals(1 + 1, 2);
}

@Nested

Used to group tests, only for non-static nested classes. Before Java 16, @BeforeAll and @AfterAll cannot be used because static methods cannot be declared on nested classes. However, it can be used if the test instance life cycle is PER_CLASS. After Java 16, this is not the case because static methods can be declared on nested classes.

import org.junit.jupiter.api.Nested;
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.assertTrue;

class NestedExampleTest {

    @Nested
    class SomeGroup {

        @Nested
        class InnerTest1_1 {
        
            @Test
            void test1() {
                assertTrue(true);
            }
        }

        @Nested
        class InnerTest1_2 {
        
            @Test
            void test2() {
                assertTrue(true);
            }
        }
    }


    @Nested
    class OtherGroup {

        @Nested
        class InnerTestA {

            @Test
            void testA() {
                assertTrue(true);
            }
        }

        @Nested
        class InnerTestB {
        
            @Test
            void testB() {
                assertTrue(true);
            }
        }
    }
}

@Timeout

An annotation that causes an exception to occur and terminate immediately if the method exceeds the specified duration, and can be annotated to a test, test factory, test template, or life cycle method. It can also be annotated to class level and applied in common to multiple methods, and the nested classes annotated @Nested are also affected. However, if set at the class level, it does not apply to the life cycle method.

In terms of description alone, the use seems to be the same as that of assertTimeoutPreemptively(), but @Timeout uses it to replace it in certain scenarios. Because assertTimeoutPreemptively() runs on a different thread than the test method, so spring transaction does not work as desired. On the other hand, @Timeout is basically used in the same thread as the test method, so it is used as an alternative if mentioned above. Thread behavior can be changed through threadMode properties.

  • INFERRED : Default value, and it follows the setting parameter or the setting file's junit.jupiter.execution.timeout.thread.mode.default property value. If not, operate as SAME_THREAD.
  • SAME_THREAD : Runs on the same thread as the test method.
  • SEPARATE_THREAD : It runs on a different thread than the test method.

unit attribute allows you to select a unit of time, and the basic unit is SECONDS.

@Test
@Timeout(value = 500, unit = TimeUnit.MILLISECONDS)
void timeoutAnnotationExample() throws InterruptedException {
    Thread.sleep(10_000);
}

@TempDir

Used to create and cleanup temporary directories. You can annotate to a parameter or field-level Path or File type.

@Test
void tempDirExample(@TempDir Path source, @TempDir Path target) throws IOException {
    Path sourceFile = source.resolve("test.txt");
    Files.write(sourceFile, "a,b,c".getBytes());

    Path targetFile = Files.copy(sourceFile, target.resolve("test.txt"));

    assertNotEquals(sourceFile, targetFile);
    assertEquals(singletonList("a,b,c"), Files.readAllLines(targetFile));
}

cleanup attribute allow you to specify cleanup actions, and there are four modes: DEFAULT, ALWAYS, ON_SUCCESS and NEVERDEFAULT mode follows the setting parameter or the setting file's junit.jupiter.tempdir.cleanup.mode.default property value, and operates as ALWAYS mode if there is no value.

Extension Model

Declarative Extension Registration

You can register one or more extensions using @ExtendWith.

@ExtendWith({ DatabaseExtension.class, WebServerExtension.class})
class ExtensionModelTest { }
// or 
@ExtendWith(DatabaseExtension.class)
@ExtendWith(WebServerExtension.class)
class ExtensionModelTest { }

Programmatic Extension Registration

@RegisterExtension is used when you need to put some value into the extension from the outside. The extended field should not be a private field or null.

If the @RegisterExtension field is static, it is registered later than the extensions registered in @ExtendWith. In this case, both the class-level callback and the method-level callback of the extension are invoked.

public class RegisterExtensionTest {

    @RegisterExtension
    static LoggingExtension staticExtension = new LoggingExtension("static version");

    @Test
    public void someTest() { }
}
Output
Type static version In beforeAll : RegisterExtensionTest
Type static version In beforeEach : someTest()

If the @RegisterExtension field is non-static, it is registered later than the extensions specified in @ExtendWith, and only invokes the method-level callback of the extension.

public class RegisterExtensionTest {

    @RegisterExtension
    LoggingExtension staticExtension = new LoggingExtension("non-static version");

    @Test
    public void someTest() { }
}
Output
Type non-static version In beforeEach : someTest()

However, if the life cycle of the test instance is PER_CLASS, it is registered before the extensions specified in @ExtendWith, and both class-level and method-level callbacks of the extension are invoked.

@TestInstance(TestInstance.Lifecycle.PER_CLASS)
public class RegisterExtensionTest {

    @RegisterExtension
    LoggingExtension staticExtension = new LoggingExtension("static version");

    @Test
    public void someTest() { }
}
Output
Type non-static version In beforeAll : RegisterExtensionTest
Type non-static version In beforeEach : someTest()
public class LoggingExtension implements BeforeAllCallback, BeforeEachCallback {

    private final String type;

    public LoggingExtension(String type) {
        this.type = type;
    }

    @Override
    public void beforeAll(ExtensionContext extensionContext) throws Exception {
        System.out.printf("Type %s In beforeAll : %s\n", type, extensionContext.getDisplayName());
    }

    @Override
    public void beforeEach(ExtensionContext extensionContext) throws Exception {
        System.out.printf("Type %s In beforeEach : %s\n", type, extensionContext.getDisplayName());
    }

    public String getType() {
        return type;
    }
}

Customizing display name

Repeatable tests, such as @RepeatedTest and @ParameterizedTest, can customize the invocation display name with the name attribute to identify each test. In this case, since the name attribute is a message format pattern, it must be used twice to display one small quotation mark. The supported placeholders are as follows.

PlaceholderDescription
{displayName}the display name of the method
{index}the current invocation index (1-based)
{arguments}the complete, comma-separated arguments list
{argumentsWithNames}the complete, comma-separated arguments list with parameter names
{0}, {1}, ...an individual argument

Example

@DisplayName("Display name of container")
@ParameterizedTest(name = "{index} ==> the rank of ''{0}'' is {1}")
@CsvSource({ "apple, 1", "banana, 2", "'lemon, lime', 3" })
void customDisplayNamesExample(String fruit, int rank) { }
