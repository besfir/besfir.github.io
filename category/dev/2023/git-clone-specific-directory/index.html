How to git clone only a specific directory

Besfir · June 7, 2023
Table of Contents
  • Let's Give it a shot
  • Initialize the local repository to which the file will be cloned
  • Enable sparse checkout config
  • Define remotes
  • Specify the directory you want to clone
  • Pull repository

Sometimes, only specific directories are needed in the git repository, and at this time, the Sparse checkout added from the v1.7 can be used.

Let's Give it a shot

For a smooth understanding, we have prepared files in the https://github.com/besfir/tutorial-sparse.git repository, and the structure is as follows.

├── .gitignore
├── README.md
└── content
    ├── dev
    │   ├── child
    │   │   ├── file_1.md           ✅
    │   │   └── file_2.md           ✅
    │   ├── file_3.md
    │   ├── file_4.md
    │   └── file_5.md
    ├── hobby
    │   ├── file_6.md               ✅
    │   ├── file_7.md               ✅
    │   └── file_8.md               ✅
    └── life
        └── file_9.md

In this repository, we will try to replicate only the content/dev/child and content/hobby directories of the main branch. If you have 5 files that you ticked when you completed the final stage, it is completed normally.

Initialize the local repository to which the file will be cloned

git init tutorial-sparse
cd tutorial-sparse

Enable sparse checkout config

git config core.sparseCheckout true

Define remotes

git remote add -f origin https://github.com/besfir/tutorial-sparse.git

Specify the directory you want to clone

echo "content/dev/child" >> .git/info/sparse-checkout
echo "content/hobby" >> .git/info/sparse-checkout

Pull repository

git pull origin main

After pulling the repository, I tried printing the structure with the tree command.

└── content
    ├── dev
    │   └── child
    │       ├── file_1.md
    │       └── file_2.md
    └── hobby
        ├── file_6.md
        ├── file_7.md
        └── file_8.md
